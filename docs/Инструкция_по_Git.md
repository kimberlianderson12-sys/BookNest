# Инструкция по работе с Git для проекта BookNest

## Слияние изменений с репозиторием

### 1. Проверка текущего состояния

```bash
# Проверить статус файлов
git status

# Посмотреть изменения
git diff

# Посмотреть список измененных файлов
git status --short
```

### 2. Добавление изменений

```bash
# Добавить все измененные файлы
git add .

# Или добавить конкретные файлы
git add static/css/style.css
git add templates/base.html
git add templates/my_reservations.html
```

### 3. Создание коммита

```bash
# Создать коммит с описанием изменений
git commit -m "feat: Улучшена мобильная адаптивность интерфейса

- Добавлено гамбургер-меню для мобильных устройств
- Таблицы преобразуются в карточки на маленьких экранах
- Оптимизированы формы и фильтры для мобильных
- Улучшена адаптивность всех компонентов интерфейса"
```

### 4. Отправка изменений в удаленный репозиторий

```bash
# Проверить удаленный репозиторий
git remote -v

# Если репозиторий еще не добавлен, добавить его:
git remote add origin https://github.com/kimberlianderson12-sys/BookNest.git

# Отправить изменения
git push origin main

# Или если используете другую ветку
git push origin <имя_ветки>
```

### 5. Если нужно обновить локальную копию перед отправкой

```bash
# Получить изменения из удаленного репозитория
git fetch origin

# Объединить изменения
git pull origin main

# Если возникли конфликты, разрешите их и затем:
git add .
git commit -m "merge: Разрешение конфликтов"
git push origin main
```

## Формат сообщений коммитов (Conventional Commits)

Рекомендуется использовать следующий формат:

```
<тип>: <краткое описание>

<подробное описание (опционально)>
```

### Типы коммитов:

- `feat:` - новая функциональность
- `fix:` - исправление ошибки
- `docs:` - изменения в документации
- `style:` - форматирование, отсутствующие точки с запятой и т.д.
- `refactor:` - рефакторинг кода
- `test:` - добавление тестов
- `chore:` - обновление задач сборки, настройки и т.д.

### Примеры:

```bash
git commit -m "feat: Добавлен поиск книг по жанру"
git commit -m "fix: Исправлена ошибка отмены бронирования"
git commit -m "style: Улучшена мобильная адаптивность интерфейса"
git commit -m "docs: Обновлена документация по установке"
```

## Работа с ветками

### Создание новой ветки:

```bash
# Создать и переключиться на новую ветку
git checkout -b feature/mobile-responsive

# Или в Git 2.23+
git switch -c feature/mobile-responsive
```

### Слияние веток:

```bash
# Переключиться на основную ветку
git checkout main

# Объединить изменения из другой ветки
git merge feature/mobile-responsive

# Удалить ветку после слияния
git branch -d feature/mobile-responsive
```

## Отмена изменений

### Отменить изменения в файле (до добавления в staging):

```bash
git checkout -- <файл>
```

### Убрать файл из staging (но сохранить изменения):

```bash
git reset HEAD <файл>
```

### Отменить последний коммит (сохранив изменения):

```bash
git reset --soft HEAD~1
```

## Просмотр истории

```bash
# Краткая история
git log --oneline

# Подробная история
git log

# История с графиком веток
git log --graph --oneline --all
```

## Теги версий

```bash
# Создать тег
git tag -a v1.0.0 -m "Версия 1.0.0 - Релиз"

# Отправить теги в удаленный репозиторий
git push origin v1.0.0

# Или отправить все теги
git push origin --tags
```

## Полезные команды

```bash
# Показать изменения в конкретном файле
git diff <файл>

# Показать статистику изменений
git diff --stat

# Показать, кто изменил файл
git blame <файл>

# Показать информацию о коммите
git show <hash_коммита>
```

## Рекомендации

1. **Делайте коммиты часто** - небольшие, логически завершенные изменения
2. **Пишите понятные сообщения** - опишите, что и зачем было изменено
3. **Перед коммитом проверяйте изменения** - используйте `git diff`
4. **Тестируйте перед push** - убедитесь, что код работает
5. **Используйте `.gitignore`** - не коммитьте временные файлы, пароли и т.д.

## Пример полного цикла работы

```bash
# 1. Проверить статус
git status

# 2. Добавить изменения
git add .

# 3. Создать коммит
git commit -m "feat: Улучшена мобильная адаптивность"

# 4. Отправить в репозиторий
git push origin main

# 5. Проверить результат на GitHub
# Откройте https://github.com/kimberlianderson12-sys/BookNest
```
